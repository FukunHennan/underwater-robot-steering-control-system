# 上位机界面截图指南

## 📸 快速截图方案

### 方案 A：手动截图（推荐，无需安装额外工具）

#### 步骤：

1. **启动上位机**
   ```bash
   cd "03-上位机软件/modbus-dashboard"
   npm run dev
   ```
   访问 `http://localhost:5175`

2. **使用浏览器内置截图功能**
   
   **Chrome/Edge 浏览器：**
   - 按 `F12` 打开开发者工具
   - 按 `Ctrl+Shift+P` 打开命令面板
   - 输入 `screenshot`
   - 选择 **"Capture full size screenshot"**（截取完整页面）
   
   **Firefox 浏览器：**
   - 按 `F12` 打开开发者工具
   - 点击右上角三个点 → **"截图"**
   - 选择 **"保存整个页面"**

3. **逐个标签页截图**
   
   依次点击以下标签并截图：
   - ✅ **姿态监控** - 显示 IMU Roll/Pitch/Yaw 数据
   - ✅ **ADC 校准** - ADC 通道校准面板
   - ✅ **PWM 控制** - PWM 频率和占空比设置
   - ✅ **舵机补偿** ⭐ - 舵机姿态自适应补偿配置（重点）
   - ✅ **GPIO 扩展** - GPIO 输入输出控制
   - ✅ **红外遥控** - NEC 协议解码参数

4. **保存位置建议**
   ```
   e:\毕业设计\Documentation\Screenshots\
   ├── 01-姿态监控.png
   ├── 02-ADC校准.png
   ├── 03-PWM控制.png
   ├── 04-舵机补偿.png  ← 最重要
   ├── 05-GPIO扩展.png
   └── 06-红外遥控.png
   ```

### 方案 B：自动化截图脚本（需要安装 Puppeteer）

#### 前提条件：

```bash
cd "03-上位机软件/modbus-dashboard"
npm install puppeteer
```

#### 运行脚本：

```bash
node screenshots.js
```

脚本会自动：
1. 启动 Chromium 浏览器
2. 访问 `http://localhost:5175`
3. 依次点击每个标签页
4. 截取完整页面并保存到 `screenshots/` 目录

#### 注意事项：
- 首次运行会下载 Chromium (~170MB)
- 确保上位机已启动 (`npm run dev`)
- 截图保存在 `03-上位机软件/modbus-dashboard/screenshots/`

## 💡 截图优化技巧

### 1. 清理干扰元素
- 关闭浏览器其他标签页
- 隐藏书签栏（Ctrl+Shift+B）
- 全屏模式（F11）后截图

### 2. 突出关键数据
- 调整浏览器缩放至 100%
- 确保所有数据清晰可见
- 避免滚动条出现在截图中

### 3. 色彩准确性
- 使用 sRGB 色彩空间
- 显示器亮度 120-150 cd/m²
- 避免反光和阴影

### 4. Mock 数据展示
如果未连接硬件，可以使用 Mock 模式：
```
http://localhost:5175?mock
```
这样可以看到模拟数据，便于截图展示。

## 🔧 故障排查

### 问题 1：页面空白
- 检查开发服务器是否运行
- 查看浏览器控制台是否有错误
- 尝试刷新页面（F5）

### 问题 2：截图不完整
- 使用 "Capture full size screenshot" 而非普通截图
- 确保页面完全加载后再截图
- 检查是否有懒加载内容

### 问题 3：颜色失真
- 确认浏览器使用 sRGB 色彩空间
- 检查显示器色彩配置文件
- 避免使用 HDR 模式
