# 水下智能转向系统 - Modbus 通信文档

## 1. 项目概述

本文档描述了水下智能转向系统的 Modbus RTU 通信协议实现，用于与上位机或其他 Modbus 主站设备进行数据交换和控制。

### 1.1 系统架构
- **主控芯片**: STM32F407VET6
- **Modbus 接口**: USART2 (PA2-TX, PA3-RX)
- **通信协议**: Modbus RTU
- **从机地址**: 0x01

## 2. 硬件配置

### 2.1 引脚分配
| 功能 | 引脚 | 说明 |
|------|------|------|
| Modbus TX | PA2 | USART2_TX |
| Modbus RX | PA3 | USART2_RX |
| 接地 | GND | 公共接地 |

### 2.2 通信参数
| 参数 | 值 | 说明 |
|------|-----|------|
| 波特率 | 9600 | 标准 Modbus 波特率 |
| 数据位 | 8 | 8位数据 |
| 校验位 | 偶校验 | Modbus 要求 |
| 停止位 | 1 | 1位停止位 |
| 从机地址 | 0x01 | 可在代码中修改 |

## 4. 支持的 Modbus 功能码

| 功能码 | 名称 | 描述 | 支持情况 |
|--------|------|------|----------|
| 0x03 | 读保持寄存器 | 读取一个或多个保持寄存器 | ✅ 支持 |
| 0x06 | 写单个寄存器 | 写入单个保持寄存器 | ✅ 支持 |
| 0x10 | 写多个寄存器 | 写入多个保持寄存器 | ✅ 支持 |

## 5. 通信示例

### 5.1 读取温度传感器数据
**请求帧**（读取地址 0x0000-0x0004，共5个寄存器）：
```
01 03 00 00 00 05 CRC16
```

**响应帧**（示例数据）：
```
01 03 0A 00 FA 00 FB 00 FC 00 FD 01 F4 CRC16
```
- 00 FA = 250 → 2.50°C
- 00 FB = 251 → 2.51°C
- 00 FC = 252 → 2.52°C
- 00 FD = 253 → 2.53°C
- 01 F4 = 500 → 5.00V

### 5.2 控制 LED 亮度
**请求帧**（设置 LED1 亮度为 50%）：
```
01 06 01 02 01 F4 CRC16
```
- 01 02 = 寄存器地址 0x0102
- 01 F4 = 500 → 50.0%

**响应帧**：
```
01 06 01 02 01 F4 CRC16
```

### 5.3 控制 GPIO 状态
**请求帧**（设置 GPIO0 为高电平）：
```
01 06 01 04 00 01 CRC16
```
- 01 04 = 寄存器地址 0x0104
- 00 01 = 1 → HIGH

**响应帧**：
```
01 06 01 04 00 01 CRC16
```

## 6. 故障排除

### 6.1 常见问题
| 问题 | 可能原因 | 解决方案 |
|------|----------|----------|
| 连接失败 | 串口线连接错误 | 检查 TX/RX 连接 |
| 无响应 | 从机地址错误 | 确认从机地址为 0x01 |
| 数据错误 | 波特率或校验位设置错误 | 确认通信参数正确 |
| CRC 错误 | 通信干扰 | 检查接线质量 |
| 寄存器地址错误 | 地址范围超出 | 检查寄存器地址范围 |

### 6.2 错误码
| 异常码 | 描述 | 原因 |
|--------|------|------|
| 0x01 | 非法功能 | 不支持的功能码 |
| 0x02 | 非法地址 | 寄存器地址超出范围 |
| 0x03 | 非法值 | 数据值超出范围 |
| 0x04 | 从机故障 | 内部处理错误 |
