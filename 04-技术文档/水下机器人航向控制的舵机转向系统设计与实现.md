# 水下机器人航向控制的舵机转向系统设计与实现

## 摘要

本设计实现了基于 STM32F407 微控制器的水下机器人航向控制系统，通过九轴惯性测量单元（IMU）实时采集姿态数据，采用卡尔曼滤波算法进行数据融合处理，并通过 PWM 信号精确控制舵机执行转向动作。系统采用 Modbus RTU 工业通信协议实现上下位机之间的稳定数据传输，上位机采用 React + TypeScript 技术栈开发 Web 监控界面，支持浏览器直连串口进行实时监控和参数配置。本系统具有响应速度快、控制精度高、界面友好等特点，可广泛应用于水下探测、海洋观测等领域。

**关键词**：水下机器人；STM32F407；舵机控制；Modbus RTU；卡尔曼滤波；Web Serial API

## 1. 绪论

### 1.1 研究背景与意义

随着海洋资源开发和深海探测技术的快速发展，水下机器人在海洋科学研究、海底资源勘探、水下工程作业等领域发挥着越来越重要的作用。航向控制作为水下机器人的核心功能之一，直接影响其运动精度和作业效率。

传统的水下机器人航向控制系统多采用专用控制器或复杂的嵌入式方案，存在成本高、扩展性差、调试困难等问题。本研究基于 STM32F407 高性能微控制器，结合现代 Web 技术，设计了一套低成本、高可靠性、易维护的水下机器人航向控制系统。

### 1.2 国内外研究现状

#### 1.2.1 水下机器人控制技术

目前国内外水下机器人控制技术主要集中在以下几个方面：
- **姿态解算算法**：互补滤波、卡尔曼滤波、扩展卡尔曼滤波等
- **控制策略**：PID 控制、模糊控制、自适应控制等
- **通信技术**：有线通信（RS485、以太网）、无线通信（水声通信、射频通信）

#### 1.2.2 存在的问题

1. **系统复杂度高**：传统方案需要多个专用芯片配合，电路复杂
2. **调试困难**：缺乏直观的监控界面，参数调整依赖经验
3. **成本高昂**：专业水下控制器价格昂贵
4. **扩展性差**：硬件固化，功能升级困难

### 1.3 本文主要研究内容

本设计主要完成以下工作：

1. **硬件平台搭建**：基于 STM32F407VET6 微控制器，集成 MS901M 九轴 IMU、舵机驱动电路、LED 指示灯等外设
2. **姿态解算算法**：实现基于卡尔曼滤波的传感器数据融合算法，提高姿态测量精度
3. **舵机控制策略**：设计高精度 PWM 控制算法，实现舵机的精确角度控制
4. **通信协议设计**：基于 Modbus RTU 协议实现上下位机通信，确保数据传输的可靠性和实时性
5. **上位机监控系统**：开发基于 React + TypeScript 的 Web 监控界面，支持浏览器直连串口
6. **系统集成与测试**：完成软硬件联调，验证系统的稳定性和控制精度

### 1.4 本章小结

本章阐述了水下机器人航向控制系统的研究背景和意义，分析了国内外研究现状及存在的问题，明确了本文的研究内容和目标。

### 2.3 技术方案选择

#### 2.3.1 微控制器选型

选用 **STM32F407VET6**，主要理由：
- Cortex-M4 内核，主频 168MHz，满足实时控制需求
- 512KB Flash + 192KB SRAM，资源充足
- 丰富的外设接口：多个 USART、TIM、ADC、GPIO
- 成熟的开发生态：HAL 库、CubeMX 工具链

#### 2.3.2 姿态传感器选型

选用 **MS901M 九轴 IMU**，包含：
- 三轴加速度计
- 三轴陀螺仪
- 三轴磁力计
- UART 接口输出，简化数据处理

#### 2.3.3 通信协议选择

采用 **Modbus RTU** 协议，优势：
- 工业标准协议，稳定性高
- 帧结构简单，易于实现
- 支持多设备组网
- 完善的错误检测机制（CRC16）

#### 2.3.4 上位机技术选型

采用 **React + TypeScript + Vite** 技术栈：
- 组件化开发，代码复用率高
- TypeScript 提供类型安全
- Vite 构建速度快，开发体验好
- Web Serial API 实现浏览器直连串口

### 2.4 本章小结

本章完成了系统的需求分析和总体方案设计，确定了以 STM32F407 为核心控制器、Modbus RTU 为通信协议、Web 技术为上位机界面的技术路线。

### 3.4 舵机驱动电路

#### 3.4.1 PWM 信号生成

使用 **TIM2** 产生 PWM 信号：
- 定时器时钟：84MHz
- 预分频系数：839
- 自动重装载值：1999
- PWM 频率：50Hz（周期 20ms）
- 脉冲宽度范围：500μs ~ 2500μs（对应 0°~180°）

**计算公式**：
```
PWM频率 = 定时器时钟 / ((预分频+1) × (重装载值+1))
        = 84MHz / (840 × 2000)
        = 50Hz

脉冲宽度 = (重装载值+1) × 占空比 × 定时器周期
0°  → 500μs  → 计数值 50
90° → 1500μs → 计数值 150
180°→ 2500μs → 计数值 250
```

#### 3.4.2 舵机供电

- 工作电压：5V
- 峰值电流：≥2A（需单独供电）
- 建议增加滤波电容（1000μF）

### 3.5 LED 指示电路

使用 **TIM3** 产生 PWM 信号控制 LED 亮度：
- 呼吸灯效果：占空比线性变化
- 状态指示：不同闪烁频率表示不同状态

### 3.6 RS485 通信接口

#### 3.6.1 MAX485 芯片电路

**接线方式**：
```

RO     →        PB11 (USART3_RX)

DE/RE  →        PB12 (方向控制)
VCC    →        5V
```

**方向控制逻辑**：
- DE=1, RE=0：发送模式
- DE=0, RE=1：接收模式

### 3.7 电源管理

**供电方案**：
- 输入：5V USB 或外部电源
- LDO 稳压：AMS1117-3.3 为 STM32 和传感器供电

### 4.4 姿态解算算法

#### 4.4.1 卡尔曼滤波原理

卡尔曼滤波是一种最优估计算法，通过预测和更新两个步骤，融合多传感器数据，得到最优状态估计。

**状态方程**：
```
x(k) = F × x(k-1) + w(k-1)
z(k) = H × x(k) + v(k)
```

其中：
- x：状态向量（角度、角速度）
- F：状态转移矩阵
- z：观测向量（传感器测量值）
- H：观测矩阵
- w：过程噪声
- v：测量噪声

#### 4.4.2 一维卡尔曼滤波实现

```c
typedef struct {
    float x;      // 状态估计值
    float p;      // 估计误差协方差
    float q;      // 过程噪声协方差
    float r;      // 测量噪声协方差
    float k;      // 卡尔曼增益
} KalmanFilter;

void Kalman_Filter_Init(KalmanFilter *kf, float q, float r)
{
    kf->x = 0.0f;
    kf->p = 1.0f;
    kf->q = q;
    kf->r = r;
    kf->k = 0.0f;
}

float Kalman_Filter_Update(KalmanFilter *kf, float measurement)
{
    // 预测
    kf->p = kf->p + kf->q;
    
    // 更新
    kf->k = kf->p / (kf->p + kf->r);
    kf->x = kf->x + kf->k * (measurement - kf->x);
    kf->p = (1 - kf->k) * kf->p;
    
    return kf->x;
}
```

#### 4.4.3 IMU 数据融合

采用互补滤波结合卡尔曼滤波：
1. **加速度计**：提供长期稳定的角度参考（低频特性好）
2. **陀螺仪**：提供短期精确的角速度信息（高频特性好）
3. **磁力计**：提供绝对航向参考

**融合公式**：
```
angle = α × (angle + gyro × dt) + (1-α) × accel_angle
```

其中 α 通常取 0.95~0.98。

### 4.5 舵机控制算法

#### 4.5.1 PWM 控制实现

```c
void Servo_Set_Angle(float angle)
{
    // 限制角度范围
    if(angle < 0) angle = 0;
    if(angle > 180) angle = 180;
    
    // 计算 PWM 脉冲宽度（计数值）
    // 0° -> 50, 180° -> 250
    uint16_t pulse = (uint16_t)(50 + (angle / 180.0f) * 200);
    
    // 设置 TIM2 CH1 比较值
    __HAL_TIM_SET_COMPARE(&htim2, TIM_CHANNEL_1, pulse);
}
```

#### 4.5.2 平滑过渡算法

为避免舵机突变，采用线性插值平滑过渡：

```c
void Servo_Smooth_Move(float target_angle, float duration_ms)
{
    float current_angle = servo_current_angle;
    float step = (target_angle - current_angle) / (duration_ms / 10);
    int steps = duration_ms / 10;
    
    for(int i = 0; i < steps; i++)
    {
        current_angle += step;
        Servo_Set_Angle(current_angle);
        HAL_Delay(10);
    }
    
    servo_current_angle = target_angle;
}
```

### 4.6 LED 控制

#### 4.6.1 呼吸灯效果

```c
void LED_Breathing(void)
{
    static uint16_t brightness = 0;
    static int direction = 1;
    
    brightness += direction * 5;
    
    if(brightness >= 100)
    {
        brightness = 100;
        direction = -1;
    }
```

## 5. 通信协议设计

### 5.1 Modbus RTU 协议概述

Modbus RTU 是 Modbus 协议的串行传输模式，特点：
- **二进制编码**：数据紧凑，传输效率高
- **CRC 校验**：16 位循环冗余校验，保证数据完整性
- **主从架构**：一个主站，最多 247 个从站

### 5.2 帧格式定义

#### 5.2.1 请求帧格式

| 字段 | 长度（字节） | 说明 |
|------|------------|------|
| 从站地址 | 1 | 0x01~0xF7 |
| 功能码 | 1 | 03/06/16 |
| 起始地址 | 2 | 高字节在前 |
| 寄存器数量/数据 | 2~N | 根据功能码变化 |
| CRC16 | 2 | 低字节在前 |

**示例：读取寄存器 0x0001~0x0003**
```
01 03 00 01 00 03 D5 CA
```
- 01：从站地址
- 03：功能码（读保持寄存器）
- 00 01：起始地址
- 00 03：寄存器数量（3个）
- D5 CA：CRC16 校验

#### 5.2.2 响应帧格式

| 字段 | 长度（字节） | 说明 |
|------|------------|------|
| 从站地址 | 1 | 与请求一致 |
| 功能码 | 1 | 与请求一致 |
| 字节数 | 1 | 后续数据字节数 |
| 寄存器数据 | N | 每个寄存器 2 字节 |
| CRC16 | 2 | 低字节在前 |

**示例：返回 3 个寄存器数据**
```
01 03 06 00 64 FF 9C 01 2C XX XX
```
- 01：从站地址
- 03：功能码
- 06：数据字节数（3×2）
- 00 64：寄存器 0x0001 = 100（Roll = 1.00°）
- FF 9C：寄存器 0x0002 = -100（Pitch = -1.00°）
- 01 2C：寄存器 0x0003 = 300（Yaw = 3.00°）
- XX XX：CRC16

### 5.3 CRC16 校验算法

```c
uint16_t Modbus_CRC16(uint8_t *data, uint16_t length)
{
    uint16_t crc = 0xFFFF;
    
    for(uint16_t i = 0; i < length; i++)
    {
        crc ^= data[i];
        for(uint8_t j = 0; j < 8; j++)
        {
            if(crc & 0x0001)
            {
                crc >>= 1;
                crc ^= 0xA001;
            }
            else
            {
                crc >>= 1;
            }
        }
    }
    
    return crc;
}
```

#### 5.4.2 Modbus 请求封装

```typescript
async function readHoldingRegisters(slaveId: number, startAddr: number, count: number) {
  // 构建请求帧
  const frame = new Uint8Array(8);
  frame[0] = slaveId;
  frame[1] = 0x03; // 功能码
  frame[2] = (startAddr >> 8) & 0xFF;
  frame[3] = startAddr & 0xFF;
  frame[4] = (count >> 8) & 0xFF;
  frame[5] = count & 0xFF;
  
  const crc = calculateCRC16(frame.slice(0, 6));
  frame[6] = crc & 0xFF;
  frame[7] = (crc >> 8) & 0xFF;
  
  // 发送请求
  await writer.write(frame);
  
  // 接收响应
  const { value } = await reader.read();
  return parseResponse(value);
}
```

### 5.5 通信异常处理

#### 5.5.1 超时机制

```c
#define MODBUS_TIMEOUT_MS 100

uint8_t Modbus_Send_Receive(uint8_t *tx_buf, uint16_t tx_len, 
                            uint8_t *rx_buf, uint16_t *rx_len)
{
    // 清空接收缓冲区
    memset(rx_buf, 0, MODBUS_BUF_SIZE);
    *rx_len = 0;
    
    // 切换到发送模式
    RS485_TX_Enable();
    
    // 发送数据
    HAL_UART_Transmit(&huart3, tx_buf, tx_len, 100);
    
    // 切换到接收模式
    RS485_RX_Enable();
    
    // 等待响应（超时保护）
    uint32_t start_time = HAL_GetTick();
    while(HAL_GetTick() - start_time < MODBUS_TIMEOUT_MS)
    {
        if(*rx_len > 0)
        {
            // 验证 CRC
            if(Modbus_Check_CRC(rx_buf, *rx_len))
            {
                return MODBUS_OK;
            }
            else
            {
                return MODBUS_CRC_ERROR;
            }
        }
        HAL_Delay(1);
    }
    
    return MODBUS_TIMEOUT;
}
```

#### 5.5.2 重试机制

```c
#define MODBUS_MAX_RETRY 3

uint8_t Modbus_Read_Registers_With_Retry(uint16_t addr, uint16_t count, uint16_t *data)
{
    for(uint8_t retry = 0; retry < MODBUS_MAX_RETRY; retry++)
    {
        uint8_t result = Modbus_Read_Registers(addr, count, data);
        if(result == MODBUS_OK)
        {
            return MODBUS_OK;
        }
        HAL_Delay(50); // 重试间隔
    }
    
    return MODBUS_COMM_ERROR;
}
```

### 6.5 问题分析与改进

#### 6.5.1 发现的问题

1. **通信偶发超时**：USB-RS485 转换器在某些电脑上存在延迟抖动
   - **解决方案**：增加超时时间至 150ms，添加重试机制

2. **舵机抖动**：在特定角度出现轻微抖动
   - **原因**：PWM 分辨率不足
   - **解决方案**：提高定时器时钟频率，增加 PWM 分辨率

3. **姿态漂移**：长时间运行后 Yaw 角出现缓慢漂移
   - **原因**：磁力计受周围磁场干扰
   - **解决方案**：增加磁力计校准功能，或使用 GPS 辅助校正

#### 6.5.2 改进建议

1. **增加 PID 控制**：实现闭环控制，提高舵机响应速度和精度
2. **优化滤波算法**：采用扩展卡尔曼滤波（EKF）或无迹卡尔曼滤波（UKF）
3. **增加无线通信**：集成 ESP32 WiFi 模块，实现远程监控
4. **增加深度传感器**：测量水下深度，完善导航信息
5. **增加防水外壳**：满足实际水下工作环境要求
