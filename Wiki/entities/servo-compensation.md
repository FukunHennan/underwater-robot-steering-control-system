# 舵机姿态补偿功能使用指南

## 📋 功能概述

舵机姿态补偿功能允许每个舵机根据IMU传感器的姿态数据（Roll/Pitch/Yaw）自动调整角度，实现自我调节和动态平衡。

---

## 🎯 核心原理

### 补偿公式

```
目标角度 = BASE_ANGLE + K_ROLL × Roll + K_PITCH × Pitch + K_YAW × Yaw
PWM脉冲宽度 = 1500 + 目标角度 × 10 (单位: μs)
```

**参数说明**:
- `BASE_ANGLE`: 基础角度偏移（度），舵机的初始位置
- `K_ROLL`: Roll角补偿系数，控制舵机对横滚角的响应强度
- `K_PITCH`: Pitch角补偿系数，控制舵机对俯仰角的响应强度
- `K_YAW`: Yaw角补偿系数，控制舵机对偏航角的响应强度

### 工作流程

1. **上位机设置系数** → 通过Modbus写入寄存器
2. **启用自动补偿** → 设置对应舵机的 AUTO_EN 标志为 1
3. **下位机实时计算** → 每轮询周期读取姿态数据并计算补偿角度
4. **自动更新PWM** → 将计算结果直接应用到PWM硬件输出

## 💻 上位机使用

### 访问路径

在 Web Dashboard 中，点击导航栏的 **"舵机补偿"** 标签页（需要在 App.tsx 中添加路由）。

### 面板功能

每个舵机补偿卡片包含：

1. **参数输入区**
   - 基础角度 (BASE_ANGLE)
   - K_Roll、K_Pitch、K_Yaw 三个系数

2. **自动补偿开关**
   - 勾选后启用该舵机的自动补偿
   - 未勾选时，舵机保持手动控制模式

3. **实时预览**
   - 显示当前姿态数据（Roll/Pitch/Yaw）
   - 计算并显示补偿后的目标角度
   - 显示对应的PWM脉冲宽度值

4. **快速预设**
   - **重置**: 所有系数归零
   - **仅 Roll**: K_Roll=1.0，其他为0
   - **Roll+Pitch**: K_Roll=0.5, K_Pitch=0.5
   - **三轴均衡**: 三个系数均为0.33

### 操作步骤

#### 示例1：让舵机1跟随Roll角同步转动

1. 打开舵机1补偿卡片
2. 设置参数：
   - BASE_ANGLE = 0
   - K_ROLL = 1.0
   - K_PITCH = 0
   - K_YAW = 0
3. 勾选"自动补偿"
4. 观察预览区的实时变化

**效果**: 当机器人Roll角为+15°时，舵机自动转到+15°位置；Roll角为-20°时，舵机转到-20°位置。

#### 示例2：多轴联动补偿

1. 设置参数：
   - BASE_ANGLE = 10（基础偏移10°）
   - K_ROLL = 0.8
   - K_PITCH = 0.5
   - K_YAW = 0.2
2. 启用自动补偿

**效果**: 
```
当 Roll=10°, Pitch=5°, Yaw=-3° 时：
目标角度 = 10 + 0.8×10 + 0.5×5 + 0.2×(-3)
         = 10 + 8 + 2.5 - 0.6
         = 19.9°
PWM = 1500 + 19.9×10 = 1699 μs
```

## 🛠️ 应用场景

### 1. 水下机器人姿态稳定

**场景**: 机器人在水流中发生倾斜，需要舵机自动调整以保持水平。

**配置**:
```
舵机1 (左舷): K_ROLL = -1.0  (反向补偿)
舵机2 (右舷): K_ROLL = +1.0  (正向补偿)
```

**原理**: 当机器人向左倾斜（Roll < 0）时，左侧舵机向上抬，右侧舵机向下压，产生恢复力矩。

### 2. 深度控制

**场景**: 通过Pitch角控制推进器角度，实现上浮/下潜。

**配置**:
```
垂直推进器: K_PITCH = 0.8
BASE_ANGLE = 0
```

**效果**: Pitch角为正（抬头）时，推进器向上偏转，帮助上浮。

### 3. 航向保持

**场景**: 根据Yaw角偏差自动调整方向舵。

**配置**:
```
方向舵: K_YAW = 1.2
BASE_ANGLE = 0
```

**效果**: Yaw角偏离设定值时，方向舵自动修正航向。

### 4. 多舵机协同

**场景**: 多个舵机配合完成复杂动作。

**配置**:
```
舵机1-4 (四角支撑): 
  K_ROLL = ±0.5, K_PITCH = ±0.5 (符号根据位置决定)
  
效果: 机器人倾斜时，四个支撑腿自动调整高度保持平台水平。
```

## ⚙️ 高级技巧

### 系数调优建议

1. **从小开始**: 初始系数设为0.1-0.3，逐步增大观察效果
2. **避免过冲**: 系数过大可能导致系统振荡
3. **考虑机械限制**: 确保补偿后的角度不超过舵机物理范围（±90°）
4. **测试不同姿态**: 在多种姿态下验证补偿效果

### 调试方法

1. **禁用自动补偿**: 先手动测试各个系数的影响
2. **单轴测试**: 一次只调整一个轴的系数
3. **观察预览**: 利用上位机的实时预览功能
4. **日志记录**: 开启Modbus通信日志，记录参数变化

### 性能优化

- **轮询频率**: 建议设置为200-500ms，平衡响应速度和通信负载
- **卡尔曼滤波**: 配合姿态数据的滤波参数，获得更平滑的补偿效果
- **死区设置**: 对于小幅度姿态变化（<1°），可忽略补偿以减少抖动

## ❓ 常见问题

### Q1: 为什么启用自动补偿后舵机不动？

**检查清单**:
- [ ] AUTO_EN 标志是否设置为 1
- [ ] 至少有一个系数不为 0
- [ ] IMU 数据是否正常读取（查看姿态页面）
- [ ] PWM 输出是否在有效范围（500-2500 μs）

### Q2: 补偿效果过于剧烈怎么办？

**解决方案**:
- 减小系数值（如从 1.0 降到 0.3）
- 增加卡尔曼滤波的 R 值（降低响应速度）
- 添加软件限幅（修改下位机代码）

### Q3: 如何保存补偿参数？

**当前状态**: 参数存储在RAM中，断电后丢失。

**未来扩展**: 可以添加到Flash存储功能，类似ADC校准参数的保存方式。

### Q4: 可以同时手动控制和自动补偿吗？

**答案**: 不可以。启用自动补偿后，手动设置的PWM值会被覆盖。如需切换，先禁用自动补偿。

## 📊 技术细节

### 下位机实现

**文件**: `02-下位机固件/F407/Drivers/BSP/Modbus/modbus.c`

**关键函数**:
```c
static uint16_t calculate_servo_with_compensation(uint8_t servo_index)
{
    // 1. 读取当前姿态数据
    float roll = modbus_get_register_float(REG_ROLL);
    float pitch = modbus_get_register_float(REG_PITCH);
    float yaw = modbus_get_register_float(REG_YAW);
    
    // 2. 读取补偿系数
    float base_angle = modbus_get_register_float(base_reg);
    float k_roll = modbus_get_register_float(base_reg + 2);
    float k_pitch = modbus_get_register_float(base_reg + 4);
    float k_yaw = modbus_get_register_float(base_reg + 6);
    
    // 3. 计算补偿角度
    float compensated_angle = base_angle + k_roll*roll + k_pitch*pitch + k_yaw*yaw;
    
    // 4. 转换为PWM并限幅
    int16_t pwm_us = (int16_t)(1500.0f + compensated_angle * 10.0f);
    if (pwm_us < 500) pwm_us = 500;
    if (pwm_us > 2500) pwm_us = 2500;
    
    return (uint16_t)pwm_us;
}
```

**执行时机**: 在 `modbus_update_sensors()` 函数末尾，每次传感器更新时执行。
