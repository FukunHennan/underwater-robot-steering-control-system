# 舵机姿态自适应补偿功能使用指南

## 📋 功能概述

舵机姿态自适应补偿功能允许每个舵机根据 IMU 姿态数据（Roll/Pitch/Yaw）自动调整角度，实现水下机器人姿态稳定控制。

**补偿公式：**
```
目标角度 = BASE_ANGLE + K_ROLL × Roll + K_PITCH × Pitch + K_YAW × Yaw
```

---

## 🚀 快速开始

### 1. 启动上位机

```bash
cd "03-上位机软件/modbus-dashboard"
npm run dev
```

访问 `http://localhost:5173`

### 2. 连接设备

1. 点击顶部导航栏的 **"连接串口"** 按钮
2. 选择正确的 COM 端口（USB转RS485）
3. 确认波特率为 **9600**，从站地址为 **0x01**

### 3. 进入补偿页面

点击顶部标签导航中的 **"补偿"** 标签（Target 图标）

## 🎯 界面说明

### 顶部信息区

- **当前姿态显示**：实时显示 Roll、Pitch、Yaw 三个轴的角度值
- **刷新参数**：从下位机读取最新的补偿参数
- **保存到 Flash**：将当前所有参数永久保存到 STM32 Flash

### 舵机配置卡片（共 8 个）

每个舵机卡片包含：

#### 1. 基础参数
- **基准角度 (BASE)**：舵机的初始角度（°）
- **K_ROLL**：Roll 轴的补偿系数
- **K_PITCH**：Pitch 轴的补偿系数
- **K_YAW**：Yaw 轴的补偿系数

#### 2. 实时预览
- 显示根据当前姿态计算出的**目标角度**
- 当倾斜开发板时，数值会实时更新

#### 3. 自动补偿开关
- **启用**：舵机会根据姿态自动调整
- **禁用**：舵机保持固定角度

#### 4. 快捷预设
- **水平模式**：所有系数归零，仅保留基准角度
- **横滚优先**：K_ROLL = 1.0，其他为零
- **俯仰优先**：K_PITCH = 1.0，其他为零
- **全向补偿**：所有系数均为 0.5

## 🔧 使用步骤

### 步骤 1：设置基准角度

1. 将机器人放置在水平位置
2. 手动调整舵机到期望的中立位置
3. 记录当前 PWM 值对应的角度
4. 在 **基准角度** 输入框中填入该值
5. 按 Enter 或点击外部应用

### 步骤 2：调整补偿系数

#### 方法 A：手动输入
直接在各系数的输入框中输入数值：
- **正系数**：姿态正向倾斜时，舵机同向转动
- **负系数**：姿态正向倾斜时，舵机反向转动
- **零系数**：该轴不影响舵机

#### 方法 B：使用预设
点击快捷预设按钮快速设置常用配置

### 步骤 3：测试效果

1. **启用自动补偿**：打开开关
2. **倾斜开发板**：观察舵机是否跟随姿态变化
3. **查看实时预览**：确认计算角度符合预期
4. **微调系数**：根据实际效果调整系数值

### 步骤 4：保存参数

确认所有舵机配置正确后，点击顶部的 **"保存到 Flash"** 按钮。

⚠️ **重要**：未保存的参数在断电后会丢失！

## 💡 应用场景示例

### 场景 1：水平稳定平台

**目标**：无论机器人如何倾斜，舵机始终保持水平

**配置**：
- BASE = 90°（垂直向上）
- K_ROLL = -1.0（反向补偿 Roll）
- K_PITCH = -1.0（反向补偿 Pitch）
- K_YAW = 0（Yaw 不影响）

**效果**：机器人倾斜 30° 时，舵机会反向转动 30°，保持垂直

### 场景 2：姿态跟随

**目标**：舵机始终指向地平线

**配置**：
- BASE = 0°（水平向前）
- K_ROLL = 0（不补偿 Roll）
- K_PITCH = -1.0（反向补偿 Pitch）
- K_YAW = 0（不补偿 Yaw）

**效果**：机器人俯仰时，舵机上下摆动保持水平指向

### 场景 3：多舵机协同

**目标**：多个舵机配合实现复杂动作

**配置示例**（4 个舵机）：
- 舵机 1：K_ROLL = 0.5, K_PITCH = 0.3
- 舵机 2：K_ROLL = -0.5, K_PITCH = 0.3
- 舵机 3：K_ROLL = 0.5, K_PITCH = -0.3
- 舵机 4：K_ROLL = -0.5, K_PITCH = -0.3

**效果**：实现差动转向或波浪运动

## ⚙️ 高级技巧

### 1. 系数调优方法

**逐步逼近法**：
1. 先将所有系数设为 0
2. 单独调整 K_ROLL，观察 Roll 轴响应
3. 单独调整 K_PITCH，观察 Pitch 轴响应
4. 最后微调 K_YAW（如果需要）
5. 反复迭代直到满意

**经验值参考**：
- 轻微补偿：0.1 ~ 0.3
- 中等补偿：0.5 ~ 0.8
- 完全补偿：1.0 ~ 1.5
- 过补偿：> 1.5（可能导致振荡）

### 2. 调试建议

✅ **推荐做法**：
- 先在静态环境下测试单个舵机
- 使用小幅度倾斜（±10°）验证响应方向
- 逐步增大倾斜角度测试线性度
- 记录最佳系数组合

❌ **避免操作**：
- 不要同时修改多个系数
- 不要在剧烈运动中调整参数
- 不要忘记保存就断电
- 不要设置过大的系数（> 2.0）

### 3. 故障排查

| 问题 | 可能原因 | 解决方法 |
|------|---------|---------|
| 舵机不动作 | 自动补偿未启用 | 检查开关是否打开 |
| 舵机反向运动 | 系数符号错误 | 将系数取反（正变负） |
| 舵机振荡 | 系数过大 | 减小系数值 |
| 响应延迟 | 通信波特率低 | 检查 RS485 连接质量 |
| 参数丢失 | 未保存到 Flash | 点击"保存到 Flash"按钮 |

## ❓ 常见问题

**Q: 为什么舵机反应很慢？**  
A: 检查 RS485 通信质量，确保波特率设置为 9600，线缆长度不超过 10 米。

**Q: 可以只补偿一个轴吗？**  
A: 可以，将不需要补偿的轴系数设为 0 即可。

**Q: 系数可以是负数吗？**  
A: 可以，负数表示反向补偿。例如 K_ROLL = -1.0 表示 Roll 正向倾斜时舵机反向转动。

**Q: 保存后重启参数还在吗？**  
A: 是的，保存到 Flash 后参数会永久存储，除非重新写入或复位。

**Q: 如何恢复默认设置？**  
A: 将所有系数设为 0，基准角度设为 90，然后保存到 Flash。
